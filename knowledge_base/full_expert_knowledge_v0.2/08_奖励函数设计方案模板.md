# 08_奖励函数设计方案模板

## 1. 作用

奖励函数设计方案是写 reward 代码前必须输出的中间产物。

它要求 Agent 先说明：为什么选这些奖励骨架、每个奖励项的作用是什么、有什么风险、训练后看什么指标。

## 3. 示例：导航任务

```yaml
奖励函数设计方案:
  reward_version: reward_v1
  task_understanding:
    任务类型: navigation_goal_reaching
    主目标: 到达目标点
    约束:
      - 不能碰撞
      - 不能越界
    偏好:
      - 尽快到达
    主要困难:
      - 终点奖励可能稀疏

  reward_complexity:
    level: Level 3
    reason: 有明确目标，但需要过程引导缓解稀疏奖励

  selected_skeletons:
    - skeleton_id: terminal_success_reward
      中文名称: 终点成功奖励
      role: task_objective
      reason: 明确告诉智能体什么是成功
      required_variables:
        - success_flag
      expected_effect: 提高成功率
      risk: 如果单独使用会稀疏

    - skeleton_id: progress_delta_reward
      中文名称: 增量进步奖励
      role: process_guidance
      reason: 引导智能体逐步接近目标
      required_variables:
        - previous_distance_to_goal
        - current_distance_to_goal
      expected_effect: 加快学习
      risk: 可能目标附近震荡

  reward_formula:
    symbolic: r = w_success * I[success] + w_p * (d_prev - d_curr) - w_time - w_collision * I[collision]

  potential_reward_hacking:
    - risk_id: goal_oscillation_exploit
      description: 智能体可能在目标附近震荡刷 progress
      severity: medium
      mitigation: 成功后终止，增加 terminal success，progress clip
      required_test: 检查目标附近轨迹
```
