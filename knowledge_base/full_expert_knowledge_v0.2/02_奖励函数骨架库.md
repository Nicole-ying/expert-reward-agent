# 02_奖励函数骨架库

## 使用说明

每个奖励骨架都应被视为一个可调用的“奖励积木”。一个完整奖励函数可以是单一骨架，也可以是多个骨架组合：

```math
R_t = \sum_i w_i r_i(s_t,a_t,s_{t+1})
```

Agent 在选择骨架时必须说明：

```text
为什么选择它？
需要哪些状态变量？
有什么风险？
训练后看什么指标？
```

---

## Skeleton 1：终点成功奖励

```yaml
skeleton_id: terminal_success_reward
中文名称: 终点成功奖励
功能角色:
  - 任务目标奖励
数学形态: r_success = R_success * I[success]
适用场景:
  - 到达目标
  - 完成任务
  - 抓取成功
  - 游戏胜利
需要变量:
  - success flag
风险:
  - 奖励稀疏
  - 随机探索难以触发
常见失败模式:
  - sparse_reward_no_learning
修正:
  - 加过程引导奖励
  - 加阶段性事件奖励
  - 加势能塑形
调用规则:
  - 如果任务有明确成功条件，优先加入
```

## Skeleton 2：失败惩罚

```yaml
skeleton_id: terminal_failure_penalty
中文名称: 失败惩罚
功能角色:
  - 安全约束惩罚
数学形态: r_failure = -R_failure * I[failure]
适用场景:
  - 摔倒
  - 碰撞
  - 越界
  - 死亡
  - 任务失败
需要变量:
  - failure flag
  - termination reason
风险:
  - 太小则智能体不怕失败
  - 太大则智能体过度保守
修正:
  - 区分轻微违规和严重失败
  - 使用连续惩罚或门控奖励
调用规则:
  - 如果环境有 failure/terminated 条件，必须考虑
```

## Skeleton 3：每步时间惩罚

```yaml
skeleton_id: time_penalty
中文名称: 每步时间惩罚
功能角色:
  - 行为效率约束
数学形态: r_time = -lambda_time
适用场景:
  - 希望尽快完成任务
  - 导航
  - 路径规划
  - 迷宫
风险:
  - 太大可能导致冒险行为
  - 对需要等待的任务可能有害
修正:
  - 使用较小权重
  - 与成功奖励配合
调用规则:
  - 如果目标是尽快完成任务，加入
```

## Skeleton 5：距离型密集奖励

```yaml
skeleton_id: distance_reward
中文名称: 距离型密集奖励
功能角色:
  - 过程引导奖励
数学形态: r_distance = -d(s_t, goal)
适用场景:
  - 目标位置明确
  - 可以定义距离
  - 成功奖励稀疏
需要变量:
  - 当前状态位置
  - 目标位置
风险:
  - 接近目标但不完成任务
  - 距离函数不等于真实任务进度
修正:
  - 加终点成功奖励
  - 加时间惩罚
  - 改成增量进步奖励
调用规则:
  - 如果可定义距离且稀疏奖励难学，可加入
```

## Skeleton 6：增量进步奖励

```yaml
skeleton_id: progress_delta_reward
中文名称: 增量进步奖励
功能角色:
  - 过程引导奖励
数学形态: r_progress = d(s_t, goal) - d(s_{t+1}, goal)
适用场景:
  - 导航
  - 接近目标
  - 机器人移动
  - 机械臂接近物体
需要变量:
  - 上一步距离
  - 当前距离
风险:
  - 目标附近震荡刷分
  - 只优化局部进展
常见失败模式:
  - goal_near_oscillation
修正:
  - 加终点奖励
  - 加时间惩罚
  - 裁剪 progress
调用规则:
  - 如果任务有“越来越接近目标”的结构，优先考虑
```

## Skeleton 8：事件奖励

```yaml
skeleton_id: event_reward
中文名称: 事件奖励
功能角色:
  - 阶段性目标奖励
数学形态: r_event = c * I[event]
适用场景:
  - 到达检查点
  - 抓到物体
  - 接触目标
  - 完成子阶段
风险:
  - 事件可重复触发刷分
  - 阶段奖励导致卡阶段
常见失败模式:
  - event_reward_farming
修正:
  - 事件只触发一次
  - 使用阶段状态机
  - 提高最终成功奖励
调用规则:
  - 如果任务可分阶段，考虑加入
```

## Skeleton 9：组件式加权奖励

```yaml
skeleton_id: weighted_sum_reward
中文名称: 组件式加权奖励
功能角色:
  - 多目标组合
数学形态: r = sum_i w_i * r_i
适用场景:
  - 连续控制
  - 多目标任务
  - 机器人
  - 自动驾驶
风险:
  - 权重难调
  - 辅助奖励压过主任务
  - 安全约束被抵消
修正:
  - 记录 reward_terms
  - 做归一化
  - 做消融
  - 安全项使用门控或大惩罚
调用规则:
  - 如果任务包含多个目标/约束，使用
```

## Skeleton 10：门控/层级奖励

```yaml
skeleton_id: gated_reward
中文名称: 门控/层级奖励
功能角色:
  - 安全优先结构
数学形态:
  if unsafe:
      r = -R_large
  else:
      r = r_task
适用场景:
  - 安全优先
  - 自动驾驶
  - 医疗控制
  - 碰撞不可接受
风险:
  - 太保守
  - unsafe 判断过宽导致学习困难
修正:
  - 区分硬约束和软约束
  - 硬约束门控，软约束连续惩罚
调用规则:
  - 如果某个约束不能被其他奖励抵消，使用
```

## Skeleton 14：势能塑形奖励

```yaml
skeleton_id: potential_based_shaping
中文名称: 势能塑形奖励
功能角色:
  - 过程引导奖励
  - 奖励塑形
数学形态:
  F(s,a,s') = gamma * Phi(s') - Phi(s)
  R' = R + F
适用场景:
  - 稀疏奖励
  - 可以定义状态好坏函数 Phi
风险:
  - Phi 设计错误会误导学习
  - 塑形项压过原始任务奖励
修正:
  - 让 Phi 与真实目标一致
  - 记录 shaping term
文献依据:
  - Ng et al. 1999 potential-based reward shaping
调用规则:
  - 如果希望提供学习引导且尽量保持原目标，考虑
```

## Skeleton 15：内在探索奖励

```yaml
skeleton_id: intrinsic_exploration_reward
中文名称: 内在探索奖励
功能角色:
  - 探索奖励
数学形态: R_total = R_ext + beta * R_int
适用场景:
  - 外部奖励非常稀疏
  - 随机探索难以成功
  - 状态空间大
风险:
  - 只探索不完成任务
  - 内在奖励压过外部奖励
常见失败模式:
  - exploration_without_task_completion
修正:
  - beta 衰减
  - 提高外部成功奖励
  - 限制内在奖励范围
调用规则:
  - 如果稀疏探索严重，考虑加入
```

## Skeleton 16：动态/课程奖励

```yaml
skeleton_id: dynamic_curriculum_reward
中文名称: 动态/课程奖励
功能角色:
  - 训练调度
数学形态: r_t = sum_i w_i(t) * r_i
适用场景:
  - 初期太难
  - 需要先学基础技能
  - 后期回归真实目标
风险:
  - 训练阶段切换不稳定
  - 解释性下降
修正:
  - 记录每个阶段
  - 做课程策略消融
调用规则:
  - 不作为第一版默认奖励，除非任务明显需要课程
```

## Skeleton 17：偏好/学习型奖励

```yaml
skeleton_id: learned_preference_reward
中文名称: 偏好/学习型奖励
功能角色:
  - 学习型奖励模型
数学形态: R_theta(s,a) 或 R_theta(tau)
适用场景:
  - 目标难以手写
  - 人类容易判断好坏
  - LLM/RLHF
  - 审美/自然性任务
风险:
  - 奖励模型本身会被利用
  - 数据偏差
  - 解释性较弱
修正:
  - 人类反馈校验
  - reward model validation
  - 约束 KL 或策略变化
调用规则:
  - 当手写奖励无法表达真实偏好时考虑
```

## 快速选择表

| 任务需求 | 优先骨架 |
|---|---|
| 明确成功 | terminal_success_reward |
| 明确失败 | terminal_failure_penalty |
| 尽快完成 | time_penalty |
| 保持不失败 | alive_bonus |
| 接近目标 | distance_reward / progress_delta_reward |
| 向前移动 | forward_progress_reward |
| 连续动作 | energy_penalty / action_smoothness_penalty |
| 安全优先 | gated_reward / constraint_penalty |
| 稀疏探索 | potential_based_shaping / intrinsic_exploration_reward |
| 多目标 | weighted_sum_reward |
| 难以手写目标 | learned_preference_reward |
