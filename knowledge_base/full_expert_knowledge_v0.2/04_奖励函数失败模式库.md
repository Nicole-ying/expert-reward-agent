# 04_奖励函数失败模式库

## 1. 使用目的

训练失败时，不能只看 mean return。Agent 需要根据行为现象、reward_terms、动作统计、成功率和失败原因，反推出奖励函数可能哪里出了问题。

诊断流程：

```text
训练日志 → 行为现象 → 匹配失败模式 → 找可能原因 → 修改奖励项 → 记录经验
```

---

## Failure Mode 1：智能体原地不动

```yaml
failure_mode_id: agent_stands_still
中文名称: 智能体原地不动
触发条件:
  - action_magnitude 很低
  - progress_reward 接近 0
  - episode_length 较长
  - success_rate 较低
可能原因:
  - alive_bonus 太高
  - energy_penalty 太高
  - action_smoothness_penalty 太高
  - progress_delta_reward 太弱
  - terminal_success_reward 太稀疏
需要查看的日志:
  - reward_terms.alive
  - reward_terms.energy
  - reward_terms.smoothness
  - reward_terms.progress
  - action_magnitude
  - distance_to_goal
修改建议:
  - 降低 alive 权重
  - 降低 energy/smoothness 权重
  - 增加 progress 权重
  - 加阶段奖励或势能塑形
  - 检查 success reward 是否足够大
```

## Failure Mode 2：智能体猛冲然后失败

```yaml
failure_mode_id: agent_moves_fast_but_falls
中文名称: 智能体猛冲然后失败
触发条件:
  - early progress_reward 很高
  - failure_count 高
  - episode_length 短
  - action_magnitude 高
可能原因:
  - forward_progress_reward 太强
  - terminal_failure_penalty 太弱
  - 缺少 stability_penalty
  - 缺少 action_smoothness_penalty
  - 缺少 energy_penalty
需要查看的日志:
  - reward_terms.forward
  - reward_terms.failure
  - reward_terms.energy
  - reward_terms.smoothness
  - action_magnitude
  - termination_reason
修改建议:
  - 降低 forward 权重
  - 增加 fall/collision penalty
  - 加姿态稳定惩罚
  - 加动作平滑惩罚
  - 加能耗惩罚
```

## Failure Mode 3：训练 return 提高，但成功率不提高

```yaml
failure_mode_id: reward_high_success_low
中文名称: reward 高但成功率低
触发条件:
  - mean_return 上升
  - success_rate 不升
  - 真实任务指标不变
可能原因:
  - 辅助奖励压过主任务
  - proxy reward 与真实目标不一致
  - reward hacking
  - terminal_success_reward 太小
需要查看的日志:
  - reward_terms.success
  - reward_terms.progress
  - reward_terms.alive
  - reward_terms.intrinsic
  - success_rate
  - failure_rate
修改建议:
  - 提高 success reward
  - 降低辅助奖励
  - 检查 reward hacking
  - 使用 success_rate 作为模型选择指标之一
```

## Failure Mode 4：目标附近震荡

```yaml
failure_mode_id: goal_near_oscillation
中文名称: 目标附近震荡
触发条件:
  - distance_to_goal 接近阈值
  - success_rate 低
  - progress_reward 反复正负波动
  - episode_length 长
可能原因:
  - progress_delta_reward 可被来回刷分
  - terminal_success_reward 太小
  - success_condition 太严格
  - 缺少 time_penalty
需要查看的日志:
  - distance_to_goal
  - reward_terms.progress
  - success_flag
  - episode_length
修改建议:
  - 增加 terminal_success_reward
  - 放宽或修正成功条件
  - 加 time_penalty
  - 对 progress reward clip
  - 成功后立即终止 episode
```

## Failure Mode 8：事件奖励重复刷分

```yaml
failure_mode_id: event_reward_farming
中文名称: 事件奖励重复刷分
触发条件:
  - event_reward 触发次数异常高
  - 行为卡在同一阶段
  - success_rate 低
可能原因:
  - event_reward 没有限制触发次数
  - 缺少阶段状态机
  - 最终成功奖励太小
需要查看的日志:
  - event_trigger_count
  - reward_terms.event
  - stage_id
  - success_rate
修改建议:
  - 事件奖励只允许触发一次
  - 引入阶段状态机
  - 增加下一阶段奖励
  - 提高最终成功奖励
```

---

## Failure Mode 9：安全被效率奖励抵消

```yaml
failure_mode_id: unsafe_reward_tradeoff
中文名称: 安全约束被效率奖励抵消
触发条件:
  - progress/speed reward 高
  - collision/failure 频繁发生
  - return 可能不低
可能原因:
  - safety penalty 太小
  - 使用简单加权和导致安全可被抵消
需要查看的日志:
  - collision_count
  - reward_terms.progress
  - reward_terms.safety
  - failure_rate
修改建议:
  - 增加 safety penalty
  - 使用 gated_reward
  - 将严重安全事件设为 episode termination
```

## Failure Mode 10：奖励尺度爆炸

```yaml
failure_mode_id: reward_scale_explosion
中文名称: 奖励尺度爆炸
触发条件:
  - reward max/min 极端
  - 出现 NaN/Inf
  - value loss 异常
可能原因:
  - 距离/速度未归一化
  - 权重过大
  - reward term 未 clip
  - 变量单位错误
需要查看的日志:
  - reward_terms.max
  - reward_terms.min
  - total_reward_distribution
  - NaN/Inf 检查
修改建议:
  - clip reward_terms
  - 归一化距离/速度
  - 降低权重
  - 检查变量单位
```

## 快速诊断表

| 现象 | 优先匹配失败模式 | 优先修改 |
|---|---|---|
| 原地不动 | agent_stands_still | 降 alive/energy，加 progress |
| 猛冲摔倒 | agent_moves_fast_but_falls | 降 forward，加 fall/smooth |
| reward 高成功率低 | reward_high_success_low | 提高 success，降辅助项 |
| 目标附近震荡 | goal_near_oscillation | 加 terminal/time，clip progress |
| 只探索不完成 | exploration_without_task_completion | 降 intrinsic，提高 external |
| 动作抖 | jerky_action | 加 smoothness |
| 不敢动 | agent_afraid_to_move | 降 action penalty |
| 刷事件 | event_reward_farming | 限制事件触发 |
| 安全被抵消 | unsafe_reward_tradeoff | 使用 gated reward |
| reward 爆炸 | reward_scale_explosion | clip/normalize |
