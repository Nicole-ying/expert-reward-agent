# 07_环境理解卡片模板

## 1. 作用

环境理解卡片用于把原始环境代码转换为奖励函数设计所需要的语义信息。

没有环境理解卡片，不允许直接进入 reward 设计。

---

## 3. 示例：机器人行走任务

```yaml
环境理解卡片:
  env_name: BipedalWalker-like
  task_summary:
    一句话任务描述: 控制机器人稳定地向前行走，避免摔倒。
    任务类型:
      - locomotion
      - continuous_control
    是否连续控制: true
    是否稀疏奖励: false
    是否安全关键: medium
    是否多目标: true

  objective:
    主目标:
      - 向前移动
    次级目标:
      - 不摔倒
    偏好目标:
      - 动作平滑
      - 能耗低
      - 姿态稳定

  failure_condition:
    是否存在明确失败条件: true
    失败条件描述: 机身接触地面或姿态异常
    failure_types:
      - fall
      - timeout

  recommended_reward_skeletons:
    必选:
      - skeleton_id: forward_progress_reward
        reason: 主目标是向前移动
      - skeleton_id: terminal_failure_penalty
        reason: 摔倒是明确失败
      - skeleton_id: energy_penalty
        reason: 连续动作需要约束能耗
    可选:
      - skeleton_id: alive_bonus
        reason: 鼓励保持不摔倒
      - skeleton_id: action_smoothness_penalty
        reason: 降低动作抖动
      - skeleton_id: stability_penalty
        reason: 保持姿态稳定

  potential_reward_hacking:
    - risk_id: fast_then_fail_exploit
      description: 如果前进奖励过大，智能体可能猛冲后摔倒
      related_skeleton: forward_progress_reward
      severity: high
    - risk_id: stand_still_exploit
      description: 如果存活奖励过大，智能体可能原地站着不走
      related_skeleton: alive_bonus
      severity: medium

  initial_reward_complexity:
    level: Level 4
    reason: 连续控制 + 多目标 + 摔倒失败 + 动作质量要求
```

## 4. 环境理解质量检查

```yaml
环境理解质量检查:
  是否识别任务类型:
  是否识别成功条件:
  是否识别失败条件:
  是否解释 observation 含义:
  是否解释 action 含义:
  是否列出可用于奖励的变量:
  是否标注不确定变量:
  是否提出潜在奖励黑客:
  是否给出知识调用列表:
```
