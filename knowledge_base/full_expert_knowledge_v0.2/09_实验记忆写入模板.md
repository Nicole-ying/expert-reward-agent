# 09_实验记忆写入模板

## 1. 作用

实验记忆用于记录每次 reward 设计、训练、诊断和修改经验。

它不是保存完整日志，而是把日志总结成未来可复用的经验。

## 3. 示例记忆

```yaml
实验记忆:
  memory_id: bipedalwalker_reward_v2_2026_06_28
  env_name: BipedalWalker-v3
  task_type:
    - locomotion
    - continuous_control

  reward_version: reward_v2
  reward_design_summary:
    selected_skeletons:
      - skeleton_id: forward_progress_reward
        role: process_guidance
        weight: 2.0
      - skeleton_id: alive_bonus
        role: survival_objective
        weight: 0.1
      - skeleton_id: terminal_failure_penalty
        role: safety_constraint
        weight: 100.0
      - skeleton_id: energy_penalty
        role: behavior_regularization
        weight: 0.001
    formula_summary: forward + alive - fall - energy

  training_result:
    mean_return: 180.5
    success_rate: 0.12
    failure_rate: 0.78
    mean_episode_length: 240

  behavior_observation:
    summary: 智能体快速向前冲，但频繁摔倒。
    positive_behavior:
      - 能产生前进动作
    negative_behavior:
      - 姿态不稳定
      - 摔倒频繁
      - 动作幅值较大

  diagnosis:
    matched_failure_modes:
      - failure_mode_id: agent_moves_fast_but_falls
        evidence: progress 高，fall 多，episode 短
    suspected_causes:
      - forward_progress_reward 权重过高
      - 缺少 action_smoothness_penalty
      - 缺少 stability_penalty

  reusable_lesson:
    short_rule: 在 locomotion 任务中，前进奖励过强容易导致猛冲摔倒。
    applies_when:
      - task_type 包含 locomotion
      - failure_mode 为 agent_moves_fast_but_falls
```

## 4. Memory 检索规则

```text
标签过滤 + 相似度检索 + 失败模式匹配
```

优先检索字段：

```text
task_type
env_name
selected_skeletons
failure_mode_id
reward_hacking_risk
reusable_lesson
```
