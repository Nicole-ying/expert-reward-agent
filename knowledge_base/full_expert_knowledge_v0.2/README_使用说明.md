# README_使用说明

## 1. v0.2 是什么

本知识库用于支持“专家知识引导的奖励函数设计 Agent”。

它不是普通的强化学习笔记，也不是只给人看的奖励函数公式大全，而是面向 LLM/Agent 使用的结构化专家知识库。它的目标是让 LLM 在设计强化学习奖励函数时，不再只凭任务描述、observation、action 和 step 函数临时拼公式，而是能够按照专家流程完成：

```text
环境理解
→ 专家知识调用
→ 奖励骨架选择
→ 奖励设计方案
→ 奖励代码生成
→ 训练反馈分析
→ 奖励黑客检查
→ 失败诊断
→ 奖励迭代
→ 经验写入 Memory
```

## 2. v0.2 和 v0.1 的区别

v0.1 主要是人类可读的知识手册。v0.2 在 v0.1 基础上做了四类升级：

```text
1. 增加 Agent 工作流；
2. 增加环境理解卡片、奖励设计方案、实验记忆模板；
3. 给知识条目增加 metadata、tags、调用规则；
4. 增加文献依据与参考资料，方便后续写论文方法论。
```

## 5. 推荐使用方式

### 5.1 人类阅读方式

先读：

```text
00_奖励函数设计Agent工作流.md
01_奖励函数设计总原则.md
02_奖励函数骨架库.md
```

再读：

```text
03_任务类型到奖励骨架的决策树.md
04_奖励函数失败模式库.md
05_奖励黑客检查清单.md
```

最后看：

```text
07_环境理解卡片模板.md
08_奖励函数设计方案模板.md
09_实验记忆写入模板.md
```

### 5.2 LLM/Agent 使用方式

不要让 LLM 直接阅读所有文件再自由发挥。推荐使用“规则路由 + 标签检索 + 相似度检索”的混合方式。

示例：

```text
如果环境理解卡片中 task_type 包含 locomotion：
  必须调用：
  - 03_任务类型到奖励骨架的决策树.md 中的“机器人行走/运动控制类任务”
  - 02_奖励函数骨架库.md 中的 forward_progress、alive_bonus、fall_penalty、energy_penalty、action_smoothness
  - 04_奖励函数失败模式库.md 中的 agent_moves_fast_but_falls、agent_stands_still、jerky_action
  - 05_奖励黑客检查清单.md
```

## 6. 最小闭环

```text
输入环境信息
→ 生成环境理解卡片
→ 调用相关知识
→ 生成奖励设计方案
→ 检查奖励黑客
→ 生成 reward 代码
→ 训练
→ 诊断失败模式
→ 修改 reward
→ 写入实验记忆
```
