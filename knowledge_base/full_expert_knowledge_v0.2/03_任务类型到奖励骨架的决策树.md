# 03_任务类型到奖励骨架的决策树

## 1. 这个文件解决什么问题

LLM 不应该只靠“相似度检索”来决定调用哪些奖励知识。奖励函数设计更像专家诊断：先判断任务类型，再按规则调用对应奖励骨架、失败模式和检查清单。

本文件用于把“环境理解卡片”转换成“知识调用列表”。

```text
环境理解卡片
→ 任务类型判断
→ 奖励骨架路由
→ 失败模式路由
→ 奖励黑客检查路由
→ 奖励代码模板调用
```

---

## 2. 总体路由规则

Agent 先从环境理解卡片中抽取以下字段：

```yaml
routing_inputs:
  task_type:
  objective:
  success_condition:
  failure_condition:
  action_type:
  observation_semantics:
  reward_sparsity:
  safety_critical:
  continuous_control:
  multi_objective:
```

然后执行规则：

```text
如果有明确 success_condition → 必须考虑 terminal_success_reward
如果有明确 failure_condition → 必须考虑 terminal_failure_penalty
如果 action_type = continuous → 必须考虑 energy_penalty / action_smoothness_penalty
如果 reward_sparsity = high → 必须考虑 progress_delta_reward / potential_based_shaping / event_reward
如果 safety_critical = true → 必须考虑 gated_reward / safety_penalty / reward_hacking_checklist
如果 multi_objective = true → 必须考虑 weighted_sum_reward，同时检查辅助奖励是否压过主任务
```

## 3. 生存 / 平衡类任务

```yaml
route_id: survival_balance_task
中文名称: 生存/平衡类任务
触发条件:
  - objective 包含 keep_alive / survive / balance / maintain_stability
  - failure_condition 明确，例如 pole_fall、robot_fall、out_of_range
典型环境:
  - CartPole
  - 倒立摆
  - 机器人保持站立
  - 无人机悬停
推荐奖励骨架:
  必选:
    - alive_bonus
    - terminal_failure_penalty
  可选:
    - stability_penalty
    - energy_penalty
    - action_smoothness_penalty
重点失败模式:
  - agent_stands_still
  - agent_afraid_to_move
  - reward_high_success_low
必须检查:
  - stand_still_exploit
  - action_penalty_freeze
初始复杂度建议: Level 2 或 Level 3
```

专家解释：

```text
如果任务本质是“别失败”，简单存活奖励可能已经足够。
但如果动作是连续控制，必须考虑动作代价和平滑性。
如果存活奖励太高，智能体可能只学会拖时间，不主动完成任务。
```

## 4. 导航 / 到达目标类任务

```yaml
route_id: navigation_goal_reaching
中文名称: 导航/到达目标类任务
触发条件:
  - objective 包含 reach_goal / navigation / go_to_target / move_to_position
  - observation 或 info 中存在 goal_position / distance_to_goal / x,y position
典型环境:
  - 迷宫
  - 移动机器人导航
  - 小车到达目标点
推荐奖励骨架:
  必选:
    - terminal_success_reward
    - terminal_failure_penalty
    - time_penalty
  稀疏时加入:
    - distance_reward
    - progress_delta_reward
    - potential_based_shaping
  有障碍时加入:
    - collision_penalty
    - gated_reward
重点失败模式:
  - goal_near_oscillation
  - reward_high_success_low
  - event_reward_farming
必须检查:
  - goal_oscillation_exploit
  - high_reward_without_success
初始复杂度建议: Level 3
```

推荐初始公式：

```math
r_t = R_{success} I[goal] - R_{fail} I[collision] - \lambda_{time} + \lambda_p(d_t - d_{t+1})
```

专家解释：

```text
导航任务通常不能只用终点奖励，因为太稀疏。
增量奖励能告诉智能体“这一步是否更接近目标”。
但增量奖励容易导致目标附近震荡，所以必须配合成功奖励和时间惩罚。
```

## 5. 机器人行走 / 连续运动控制类任务

```yaml
route_id: locomotion_continuous_control
中文名称: 机器人行走/连续运动控制类任务
触发条件:
  - task_type 包含 locomotion / walking / running / continuous_control
  - action_space 为 Box 或连续动作
  - objective 包含 move_forward / walk / run / travel_distance
典型环境:
  - BipedalWalker
  - Ant
  - Humanoid
  - Hopper
  - Quadruped
推荐奖励骨架:
  必选:
    - forward_progress_reward
    - terminal_failure_penalty
    - energy_penalty
  常用:
    - alive_bonus
    - action_smoothness_penalty
    - stability_penalty
重点失败模式:
  - agent_moves_fast_but_falls
  - agent_stands_still
  - jerky_action
  - agent_afraid_to_move
必须检查:
  - fast_then_fail_exploit
  - stand_still_exploit
  - action_penalty_freeze
初始复杂度建议: Level 4
```

推荐初始公式：

```math
r_t = w_1 v_x + w_2 r_{alive} - w_3 I[fall] - w_4 \|a_t\|^2 - w_5 \|a_t-a_{t-1}\|^2
```

专家解释：

```text
只奖励前进会导致猛冲摔倒。
只奖励存活会导致原地站立。
能耗惩罚太大又会导致不敢动。
所以 locomotion 类任务一般需要前进、失败、能耗、平滑、稳定之间的平衡。
```

## 6. 机械臂抓取 / 操作类任务

```yaml
route_id: manipulation_grasping
中文名称: 机械臂抓取/操作类任务
触发条件:
  - task_type 包含 manipulation / robotic_arm / grasp / pick / place / lift
  - objective 包含 reach_object / grasp_object / lift / place
典型环境:
  - 机械臂抓取
  - 物体搬运
  - 插孔任务
  - 开门任务
推荐奖励骨架:
  必选:
    - terminal_success_reward
    - event_reward
    - progress_delta_reward
  常用:
    - distance_reward
    - energy_penalty
    - action_smoothness_penalty
    - gated_reward
重点失败模式:
  - event_reward_farming
  - reward_high_success_low
  - goal_near_oscillation
必须检查:
  - event_reward_farming
  - high_reward_without_success
初始复杂度建议: Level 4
```

阶段拆解：

```text
接近物体 → 接触物体 → 抓取物体 → 抬升物体 → 移动到目标 → 放置成功
```

专家解释：

```text
操作类任务往往需要阶段奖励，但阶段奖励最容易被重复刷分。
所以事件奖励最好配合阶段状态机，并且最终成功奖励必须足够重要。
```

## 7. 自动驾驶 / 安全关键任务

```yaml
route_id: autonomous_driving_safety
中文名称: 自动驾驶/安全关键任务
触发条件:
  - task_type 包含 driving / traffic / lane_keeping / safety_critical
  - safety_critical = true
典型目标:
  - 向前行驶
  - 保持车道
  - 避免碰撞
  - 遵守交通规则
  - 保持舒适性
推荐奖励骨架:
  必选:
    - progress_delta_reward
    - terminal_failure_penalty
    - gated_reward
  常用:
    - action_smoothness_penalty
    - energy_penalty
    - event_reward
    - weighted_sum_reward
重点失败模式:
  - unsafe_reward_tradeoff
  - reward_high_success_low
  - agent_stands_still
必须检查:
  - safety_tradeoff_exploit
  - high_reward_without_success
初始复杂度建议: Level 4 或 Level 5
```

专家解释：

```text
安全关键任务中，安全不能简单地被效率奖励抵消。
如果碰撞是不可接受的，就不应只用小的碰撞惩罚，而应考虑门控、层级或约束式奖励。
```

## 8. 稀疏探索类任务

```yaml
route_id: sparse_exploration
中文名称: 稀疏探索类任务
触发条件:
  - reward_sparsity = high
  - random_policy 几乎无法触发 success
  - 任务长程、状态空间大
推荐奖励骨架:
  必选:
    - terminal_success_reward
  可选:
    - event_reward
    - progress_delta_reward
    - potential_based_shaping
    - intrinsic_exploration_reward
    - dynamic_curriculum_reward
重点失败模式:
  - exploration_without_task_completion
  - event_reward_farming
  - reward_high_success_low
必须检查:
  - high_reward_without_success
  - event_reward_farming
初始复杂度建议: Level 3 到 Level 5
```

专家解释：

```text
稀疏任务中，外部奖励太少，智能体学不到。
但内在探索奖励也可能让智能体只探索不完成任务。
所以探索奖励通常需要衰减，并且必须监控 success_rate。
```

## 9. 多目标任务

```yaml
route_id: multi_objective_task
中文名称: 多目标任务
触发条件:
  - objective 包含多个目标
  - 目标之间可能冲突，如速度、安全、能耗、舒适性
推荐奖励骨架:
  必选:
    - weighted_sum_reward
  视情况加入:
    - gated_reward
    - dynamic_curriculum_reward
    - learned_preference_reward
重点失败模式:
  - reward_high_success_low
  - unsafe_reward_tradeoff
  - reward_scale_explosion
必须检查:
  - reward_terms_scale_test
  - safety_tradeoff_exploit
  - high_reward_without_success
初始复杂度建议: Level 4 或 Level 5
```

## 10. 反向路由：根据训练失败现象调用诊断知识

| 训练现象 | 调用失败模式 | 优先检查奖励项 |
|---|---|---|
| 原地不动 | agent_stands_still | alive、energy、progress |
| 猛冲摔倒 | agent_moves_fast_but_falls | forward、failure、smoothness |
| reward 高但成功率低 | reward_high_success_low | success、progress、alive、intrinsic |
| 目标附近震荡 | goal_near_oscillation | progress、success、time |
| 只探索不完成 | exploration_without_task_completion | intrinsic、success |
| 动作抖动 | jerky_action | smoothness、energy、action_delta |
| 不敢动 | agent_afraid_to_move | energy、smoothness、progress |
| 事件刷分 | event_reward_farming | event、stage、success |
| 安全被抵消 | unsafe_reward_tradeoff | safety、progress |
| reward 爆炸 | reward_scale_explosion | all reward_terms |
